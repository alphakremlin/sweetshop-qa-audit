Evidence Collector — Bug Report
Bug #1 — Broken Product Image (404)
Critical
Location/sweets — Wham Bars product
DetailImage filename is whan.jpg (typo). Fires a 404 on every page load. Customers see a broken image icon instead of the product.
FixRename asset file to wham.jpg and update the src reference.
Bug #2 — Legacy Google Analytics (Collecting Zero Data)
Critical
LocationAll pages — <head> tag
DetailSite uses Universal Analytics tag UA-131763802-1. Google sunset all UA properties in July 2023. No analytics data has been collected since then.
FixReplace with GA4 measurement ID (G-XXXXXXXX) from Google Analytics dashboard.
Bug #3 — No Basket Confirmation After Adding Items
Critical
Location/sweets → /basket flow
DetailAfter clicking "Add to Basket" there is no visual confirmation — no toast, no counter update, no animation. Users cannot tell if their item was added.
FixAdd a toast notification or animate the basket counter on item add.
Bug #4 — 16 Product Images Missing Alt Text
High
Location/sweets — all product images
DetailAll 16 product images lack descriptive alt attributes. Screen reader users receive no product information. Fails WCAG 1.1.1 and UK Equality Act requirements.
FixAdd descriptive alt text to every image. e.g. alt="Wham Bars - retro chewy bar, £0.15"
Bug #5 — City Dropdown Typo in Billing Form
High
Location/basket — billing address form
DetailCity dropdown contains "Swansow" — should be "Swansea". Customers selecting this city receive incorrect location data on their order.
FixCorrect spelling to "Swansea" in the dropdown option value and label.
Bug #6 — Dead Social Media Links
Medium
Location/login — footer
DetailTwitter, Facebook, and LinkedIn icons present in footer but non-functional. No href targets. Clicking does nothing.
FixAdd correct href URLs to each social icon or remove them entirely.
