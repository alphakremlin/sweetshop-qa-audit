EVIDENCE COLLECTOR — BUG REPORT
---------------------------------

BUG #1 — Broken Product Image
  Location : /sweets — Wham Bars product
  Detail   : Image filename is "whan.jpg" (typo). Should be "wham.jpg".
             Fires a 404 on every page load. Customer sees a broken image.
  Severity : HIGH
  Fix      : Rename asset file to wham.jpg and update the reference.

BUG #2 — Typo in City Dropdown
  Location : /basket — billing form
  Detail   : City option reads "Swansow". Should be "Swansea".
  Severity : MEDIUM
  Fix      : Correct spelling in dropdown option value and label.

BUG #3 — Dead Social Media Links
  Location : /login — footer social icons
  Detail   : Twitter, Facebook, LinkedIn icons present but non-functional.
             No href targets. Clicking does nothing.
  Severity : MEDIUM
  Fix      : Add correct href URLs or remove icons entirely.

BUG #4 — Broken Favicon Path
  Location : All pages
  Detail   : Favicon reference appears incomplete/broken.
             Browser tab shows no icon.
  Severity : LOW
  Fix      : Verify favicon.png exists at declared path and correct reference.

BUG #5 — Legacy Google Analytics (UA Property)
  Location : All pages
  Detail   : Site uses UA-131763802-1 (Universal Analytics).
             Google sunset UA properties in July 2023.
             No analytics data is being collected.
  Severity : HIGH
  Fix      : Replace with GA4 measurement ID (G-XXXXXXXX).

BUG #6 — Outdated Copyright
  Location : Footer — all pages
  Detail   : Reads "Sweet Shop Project 2018". Outdated by 8 years.
  Severity : LOW
  Fix      : Update to current year or use dynamic year via JS.

BUG #7 — No Basket Confirmation Feedback
  Location : /sweets → /basket flow
  Detail   : After clicking "Add to Basket" there is no visual confirmation
             to the user (no toast, no count update animation, no modal).
             User cannot tell if item was added successfully.
  Severity : HIGH
  Fix      : Add confirmation feedback — toast notification or animated
             basket counter increment.

BUG #8 — Promo Code Field — No Valid Codes Documented
  Location : /basket
  Detail   : Promo code input field present with validation message but
             no valid codes exist or are documented. Users attempting
             to use a promo code will always fail.
  Severity : MEDIUM
  Fix      : Either add valid promo codes or remove the field if unused.

BUG #9 — Empty Basket — No Confirmation Dialog
  Location : /basket
  Detail   : "Empty Basket" action link has no confirmation step.
             One accidental click clears the entire basket.
  Severity : MEDIUM
  Fix      : Add "Are you sure?" confirmation dialog before clearing basket.

ACCESSIBILITY AUDITOR — WCAG 2.1 AA REPORT
--------------------------------------------

ISSUE #1 — Missing Alt Text on Product Images
  WCAG    : 1.1.1 Non-text Content
  Detail  : All 16 product images on /sweets lack descriptive alt
            attributes. Screen reader users receive no product information.
  Severity: CRITICAL
  Fix     : Add descriptive alt text to every product image.
            Example: alt="Wham Bars - retro chewy bar sweets, £0.15"

ISSUE #2 — Social Media Icons — No Accessible Labels
  WCAG    : 1.1.1 Non-text Content
  Detail  : Footer social icons (Twitter, Facebook, LinkedIn) have no
            aria-label or visible text. Screen readers cannot identify them.
  Severity: HIGH
  Fix     : Add aria-label="Twitter" (etc.) to each icon link.

ISSUE #3 — Form Validation Errors Not Linked to Inputs
  WCAG    : 1.3.1 Info & Relationships
  Detail  : Validation error messages on login and basket forms are not
            programmatically linked to their inputs via aria-describedby.
            Screen reader users may not hear the error message.
  Severity: HIGH
  Fix     : Add aria-describedby on each input pointing to its
            error message element ID.

ISSUE #4 — Colour Contrast on Promotional Banner Not Verified
  WCAG    : 1.4.3 Contrast Minimum (4.5:1 for normal text)
  Detail  : "Sale now on" and "Get 20% off" banners could not be
            verified for contrast ratio. Requires manual check.
  Severity: MEDIUM
  Fix     : Run banner text through a contrast checker. Minimum 4.5:1.

ISSUE #5 — No Skip Navigation Link
  WCAG    : 2.4.1 Bypass Blocks
  Detail  : Keyboard users must tab through entire navigation on every
            page before reaching main content. No skip link present.
  Severity: MEDIUM
  Fix     : Add <a href="#main-content" class="skip-link">Skip to content</a>
            as first focusable element in the page.

ISSUE #6 — No Autocomplete Attributes on Login Form
  WCAG    : 1.3.5 Identify Input Purpose
  Detail  : Email and password fields on /login lack autocomplete attributes.
            Password managers and assistive technologies cannot identify fields.
  Severity: MEDIUM
  Fix     : Add autocomplete="email" and autocomplete="current-password".

ISSUE #7 — Typo in Dropdown Read by Screen Readers
  WCAG    : 3.1.1 Language of Page
  Detail  : "Swansow" will be mispronounced by screen readers.
  Severity: LOW
  Fix     : Correct spelling to "Swansea" (also listed in Bug #2).

ACCESSIBILITY SCORE: 4/10
STATUS: FAILS WCAG 2.1 AA minimum standard
